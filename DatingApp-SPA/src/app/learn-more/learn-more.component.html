The API

I created my own API and database using ASP.NET Core and Microsoft Entity Framework. For the development build, the database was done in SQLite, while the production build has the database in MySQL. There were controllers for the users, messages, photos, and the authentication which handled the HTTP get and post requests. The main tables created for the database were for users, messages, photos, and likes. The data for the first 10 users was seeded into the database so that the site would be somewhat populated. The API utilizes asynchronous methods to get the information from the database and ensures that the program can continue to run without waiting for other methods to finish. This makes for a more responsive UI and overall better performance in the application.

The Client Application

The client is made using the Angular framework. It is a single page application that uses routing to change the pages. Some pages have a route guard on them to prevent unauthorized users from accessing them as well as resolvers to get the user data before a route is activated. There is a user service which initiates the HTTP requests to the API. There is an auth service which deals with registering users, changing a user's photo, logging in a user, checking if the user is logged in, and caching the user's token into local storage. The client is made up of multiple components, some of which have parent-child relationships where the Input properties pass the data down and the Output properties pass the data back up.
